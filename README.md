# Yong Zheng's Life Operating System

中文名称：郑勇的人生操作系统  
Brand short name: Life OS

This is a bilingual, static-first personal knowledge website built with Next.js App Router, TypeScript, Tailwind CSS, and Markdown content.

## Features

- Bilingual routes: `/en/` and `/zh/`
- Static export with `next build`
- Markdown articles stored in `content/articles/zh/` and `content/articles/en/`
- Front matter metadata for title, summary, category, tags, date, language, and draft status
- Home, About, Books, Articles, Article Detail, Categories, and Tags pages
- Static JSON search index generated at build time
- RSS feed generated at `public/rss.xml`
- Sitemap generated by Next.js
- GitHub Issues powered article comments, likes, and time/popular sorting
- Vercel-compatible static site

## Requirements

- Node.js 20 or newer is recommended
- npm

## Local Development

```bash
npm install
npm run dev
```

Then open:

```text
http://localhost:3000
```

The development command first builds `public/search-index.json`, then starts the Next.js dev server.

## Static Build and Export

```bash
npm run build
```

Because `next.config.mjs` uses:

```js
output: 'export'
```

the build creates a static site in:

```text
out/
```

You can browse it with a simple static server:

```bash
npx serve out
```

You can also open `out/index.html` directly for the root language selector. For full client-side navigation and search, a static server is recommended.

## Writing Articles

Add Chinese articles here:

```text
content/articles/zh/
```

Add English articles here:

```text
content/articles/en/
```

Each article should use `.md` or `.mdx` and include front matter:

```md
---
title: "文章标题"
title_en: "English Title"
date: "2026-05-18"
summary: "摘要"
summary_en: "Summary"
category: "Life OS"
tags:
  - First Principles
  - Systems Thinking
language: "zh"
draft: false
---
```

The filename becomes the article slug. For example:

```text
content/articles/zh/life-os-introduction.md
```

becomes:

```text
/zh/articles/life-os-introduction/
```

## Search

Search uses a static index generated by:

```bash
npm run build:index
```

The index is written to:

```text
public/search-index.json
```

`npm run dev` and `npm run build` both run this step automatically.

## Comments and Likes

Article pages include a fully static interaction layer:

- Comment submission is handled by [utterances](https://utteranc.es/) and stored as GitHub Issues in `zhengyong/life-operating-system`.
- Comment and like counts are read from the public GitHub API in the browser.
- Sorting supports newest first and popular first.
- No database, backend server, or frontend token is required.

Required GitHub setup:

1. Enable Issues for `zhengyong/life-operating-system`.
2. Install the [utterances GitHub App](https://github.com/apps/utterances) for the repository.
3. Make sure the repository is public, or GitHub API reads and public commenting will not work for visitors.

The integration uses the article URL pathname as the issue title, so each language/article URL gets its own discussion thread.

## Deployment

For Vercel, import the repository and use:

```text
Build Command: npm run build
Output Directory: out
Install Command: npm install
```

Before production deployment, update `metadataBase`, `robots.ts`, `sitemap.ts`, and the RSS `siteUrl` in `scripts/build-search-index.ts` to your real domain.
